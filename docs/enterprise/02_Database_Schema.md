# RedRainbow Database Architecture

The RedRainbow platform uses a polyglot persistence strategy to optimize for different data workloads.

## 1. MySQL (Relational Core)
Handles highly structured, transactional, and relational data.

### Key Tables
- **`tenants`**: `id`, `name`, `created_at`
- **`users`**: `id`, `tenant_id`, `email`, `password_hash`, `role`, `mfa_enabled`
- **`assets`**: `id`, `tenant_id`, `asset_type` (domain, server, container, cloud), `identifier`, `criticality`, `owner`, `tags`
- **`scans`**: `id`, `tenant_id`, `asset_id`, `status`, `type`, `start_time`, `end_time`
- **`findings`**: `id`, `tenant_id`, `asset_id`, `scan_id`, `severity`, `title`, `description`, `remediation`, `status`
- **`incidents`**: `id`, `tenant_id`, `title`, `severity`, `status`, `assigned_to`, `created_at`
- **`compliance_frameworks`**: `id`, `name`, `version` (e.g., ISO 27001, SOC 2)
- **`compliance_mappings`**: `finding_id`, `framework_id`, `control_id`
- **`audit_records`**: `id`, `tenant_id`, `user_id`, `action`, `resource`, `timestamp`

## 2. MongoDB (Document & Telemetry)
Handles unstructured or highly variable schema data, logs, and raw outputs.

### Key Collections
- **`telemetry`**: Continuous streams from Monitorix SDK (CPU/RAM, network connections, file integrity events).
- **`raw_scan_results`**: Full JSON outputs from security agents (e.g., raw Nmap XML/JSON, ZAP reports, Trivy outputs).
- **`agent_data`**: Health, status, and configuration details for the containerized security workers.
- **`evidence_metadata`**: Metadata, hashes, and links to PCAP files, memory dumps, and malware samples stored in the Evidence Vault (S3/MinIO).

## 3. OpenSearch (SIEM & Logs)
Handles high-volume time-series data for searching, dashboarding, and SIEM correlation.

### Key Indices
- **`security-events-*`**: Ingested endpoint logs, firewall logs, server logs, and application logs.
- **`siem-alerts-*`**: Correlated alerts generated by the SIEM detection engine based on `security-events`.
- **`threat-intel-*`**: Ingested IOCs (IPs, hashes, domains) from MISP, OTX, and other feeds for fast correlation.

## 4. Redis (Cache & Event Bus)
Handles ephemeral data and messaging.

### Key Usage
- **`Cache`**: Frequently accessed data like user sessions, JWT blacklists, and asset metadata.
- **`Task Queue`**: Celery broker for distributing scan jobs to the Security Agent Mesh.
- **`Realtime Events`**: Pub/Sub channels for WebSocket streams pushing live telemetry to the Mission Control UI.
